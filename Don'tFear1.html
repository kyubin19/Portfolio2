Don't Fear the OOP

Here is a simple introduction to object-oriented programming, one of the major concepts of programming in Java. Although this tutorial explains OOP mostly through analogy, bolded words are the actual syntax used by Java programmers, so keep a look out for them!Hopefully this will help ease you into the nature of Java, and after reading this short tutorial, all of you will have a good grasp on object-oriented programming!There are three different levels in this tutorial which are coded by colour.

Green is for the readers who are unsure what object-oriented programming is and want the most simplest and basic introduction.

Yellow is for those who are not ready to learn the details in Java code itself, but still want to be able to understand object-oriented programming in order to be able to read and follow it.  

Last but not least, red is for the eager new coders who want to slowly ease into the basics of being able to read and write code in Java.

Meet Gloria. Gloria writes children’s books about animals. Her readers are never disappointed, because they know that Gloria will write consistent and interesting books. How does she manage this? Before Gloria writes anything, she sits down at her desk and places a single piece of paper where she will fit her whole story down on her desk. On every page, Gloria includes a sequence of events, or plot. Everything else such as setting, character development, and how her characters interact is all taken care of elsewhere. Let's say that she has just begun writing "Monkey Escape at the Toronto Zoo", her fifty-ninth book. The first and foremost thing that Gloria requires is a setting. So, she pulls down a binder she has marked "Zoo", opens to the first page, and reads the first few lines:

 

Every zoo has a few key ingredients: a location, animals, a name, monkeys, and workers. A standard zoo would have ten animals and two monkeys.

This description of a zoo did two very important things: established the key ingredients for a zoo and set some default values that would occur in a zoo.

          number of animals = 10
          number of monkeys = 2
 

Although this style of writing may be quite odd it is rather straightforward to Gloria. The first line is simply allowing Gloria to define a zoo. If this name was perhaps two or more words (for e.g. TorontoZoo) she would not use a space to separate the words and would capitalize the first letter in each word. This is to prevent any confusion for her editor in Chicago.  

The following step for Gloria would be to declare the variables that a class of zoos might have. Variables are things which define how the zoo looks such as workers, animals, monkeys, etc. The last step to be taken is to construct a standard zoo with default values for her variables (for e.g. ten animals, two monkeys, etc.)

That would be the process of declaring each variable name and value. Some common variable types include "int" s and "String" for a string of letters.  

In Gloria’s style of writing, the semicolon can be seen as a period, as it marks the end of a statement. Last but not least, a curly brace, “{“ is used to group a set of statements. This is essential for her editor to know where each set of statements begins and ends.All classes will end with a "}".

(Gloria will send all necessary binders with her manuscript). Her editor will find out the proper amounts of each element in the constructor..  

After declaring all of the variables, that’s where the constructor is found. It simply begins with public Zoo(). The statements after the method header line set the default values for her variables. In other words, how Gloria’s zoo object would look like if it was instantiated without specifying any of its values.

Now that the main event has occurred, Gloria wants to let her readers know what is going on. To do so, she uses the "System.out.println" statement. This lets her editor know to put all the text she puts in between quotation marks into the final copy of the book. Why is “this.whatIsYourName” and “riend.whatIsYourName” then not in quotation marks? Simply because they’re not literals, but rather an object's method. By keeping it out of the quotation marks, Gloria lets her editor know that he has to go look for the friend object (he would find out that it was of the "Monkey" class) then for the method "whatIsYourName" (he would see that it returns its name). All of that for a name!

As for “this”, it refers to the calling object. Every method needs to be called by an object, which in this case would be the monkey that is escaping from its cage.

Had she put this.whatIsYourName or friend.whatIsYourName inside of the quotation marks, her editor would have printed out in the final copy of the book: "this.whatIsYourName() + “has run away with " + friend.whatIsYourName()". In order for her editor to substitute the variable name for the word name, Gloria doesn’t include them in the quotation marks. The plus sign "+" tells her editor to join the two statements.

You may be wondering what is so special about this story? Well, it's not everything that's happening in the main routine, but rather everything that happens behind the scenes. When Gloria created that zoo called Toronto Zoo, she created (instantiated) an object of type Zoo. This object had all the characteristics of a standard Zoo. The same applies for the Alex object and the Julian object. While some of the traits of these objects were specified (initialized) at the time of creation, most of their traits were specified back in the binders (classes). This allowed Gloria to create a lot of character development while only saying a little bit in her main plot.
In the end, Gloria’s main plot turns out to be nothing more than a collection of references to objects, which in turn are references to classes. 

So what are the benefits of writing in such a manner? This is much more efficient because if Gloria decided that she wanted Julian to eat less fruits. Instead of going back and laboriously changing every minor detail, Gloria can just make a few alterations to the Julian object at the time of instantiation. She's got a lot of options, but the point is that she only needs to make the changes in one place.

Who is the editor?

There have been many references to Gloria’s editor in Chicago, but who is he and what does he do? He takes all of Gloria’s scripts, reads them, identifies all of her errors, and then returns them. On the. computer, this is known as the compiler. All Java programs must be compiled before they can be run. The compiler will tell you everything that you did wrong, time and time again.
